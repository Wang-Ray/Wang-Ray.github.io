## Payment Facilitator (PayFac/PF) 详细介绍

### 1. 定义与角色

**Payment Facilitator（支付服务商）** 是经卡组织授权，以自身名义作为主商户（Master Merchant）与收单机构签约，并在其账户下为子商户（Sub-Merchant）提供支付服务的机构。

```
PayFac = 主商户 + 支付服务提供商 + 风险承担者
```

### 2. PayFac 的核心特征

| 特征           | 说明                                      |
| -------------- | ----------------------------------------- |
| **主商户身份** | 拥有自己的 MID，子商户交易通过主 MID 处理 |
| **全栈服务**   | 提供从入驻到结算的完整支付服务            |
| **风险承担**   | 对子商户的欺诈、退单等承担全部责任        |
| **资金托管**   | 资金先到 PayFac，再分发给子商户           |
| **快速入驻**   | 子商户可在分钟内完成入驻并开始收款        |

**准备金策略:**

```
Reserve (准备金) 计算:

1. Rolling Reserve (滚动准备金)
   - 扣留每笔交易的 5-10%
   - 180天后释放
   
2. Upfront Reserve (前置准备金)
   - 入驻时一次性缴纳
   - 通常为月交易额的 50-100%
   
3. Capped Reserve (封顶准备金)
   - 累计到一定金额后停止扣留
   - 如: 最高 $50,000

风险等级与准备金:
```

### 6. PayFac 收入模式

#### 6.1 收入来源

```
PayFac 收入结构:
┌─────────────────────────────────────────────────────────────┐
│              商户支付总费率: 2.9% + $0.30                     │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│  Interchange  │    │  Assessment   │    │   PayFac      │
│   + Network   │    │    Fees       │    │   Margin      │
│   ~2.0%       │    │   ~0.15%      │    │   ~0.75%      │
│               │    │               │    │   + $0.30     │
│   → Issuer    │    │   → Networks  │    │   → PayFac    │
└───────────────┘    └───────────────┘    └───────────────┘
```

#### 6.2 收入计算示例

```
假设:
- 月处理交易量: 1,000,000 笔
- 平均交易金额: $50
- 月交易总额: $50,000,000
- 商户费率: 2.9% + $0.30
- PayFac 成本 (Buy Rate): 2.15% + $0.10

收入计算:
┌─────────────────────────────────────────────────────────────┐
│ 总收费: $50M × 2.9% + 1M × $0.30 = $1,450,000 + $300,000   │
│        = $1,750,000                                         │
│                                                             │
│ 成本:   $50M × 2.15% + 1M × $0.10 = $1,075,000 + $100,000  │
│        = $1,175,000                                         │
│                                                             │
│ 毛利:   $1,750,000 - $1,175,000 = $575,000/月              │
│                                                             │
│ 毛利率: 32.9%                                               │
└─────────────────────────────────────────────────────────────┘
```

### 9. PayFac vs ISO 关键决策因素

```
选择 PayFac 模式的信号:
├── ✅ 平台型业务模式（电商平台、SaaS、Marketplace）
├── ✅ 需要快速商户入驻（分钟级）
├── ✅ 有技术团队能力
├── ✅ 愿意承担风险换取更高收益
├── ✅ 需要深度定制支付体验
└── ✅ 计划全球扩张

选择 ISO 模式的信号:
├── ✅ 传统销售型业务
├── ✅ 缺乏技术开发能力
├── ✅ 不愿承担支付风险
├── ✅ 专注于客户获取和服务
└── ✅ 启动资金有限
```

### 11. 总结

**PayFac 模式的核心价值主张:**

```
"通过技术能力和风险承担，为平台型业务提供快速、无缝的支付服务，
获取更高的交易利润和更深的商户关系"
```

**关键成功因素:**

1. **技术能力** - 自建或整合完整的支付技术栈
2. **风控能力** - 有效管理子商户风险，控制损失
3. **合规能力** - 满足卡组织和监管要求
4. **运营能力** - 高效的商户入驻和客户服务
5. **规模效应** - 交易量越大，单位成本越低
