## ISO (Independent Sales Organization) 详细介绍

### 1. 定义与角色

**ISO（独立销售组织）** 是经卡组织（Visa/Mastercard）和收单机构授权，代表收单机构向商户销售支付服务的第三方机构。

```
ISO 本质上是一个"销售渠道"，而非支付处理者
```

### 2. ISO 的分类

#### 按注册类型分

| 类型                   | 说明             | 特点                                 |
| ---------------------- | ---------------- | ------------------------------------ |
| **Registered ISO**     | 在卡组织正式注册 | 可以直接与收单机构签约，品牌可见     |
| **Non-Registered ISO** | 未在卡组织注册   | 必须通过Registered ISO或收单机构运营 |

#### 按业务模式分

| 类型               | 说明                    |
| ------------------ | ----------------------- |
| **Retail ISO**     | 专注线下POS终端销售     |
| **E-commerce ISO** | 专注线上支付网关销售    |
| **Wholesale ISO**  | 批发模式，发展下级Agent |
| **Hybrid ISO**     | 混合模式，线上线下兼顾  |

### 4. ISO 的核心职责

#### 4.1 商户获取 (Merchant Acquisition)
```
├── 市场营销和销售
├── 商户资质审核（初审）
├── 定价方案制定
├── 合同签署（代表收单机构）
└── 竞争对手商户转化
```

#### 4.2 商户入驻 (Merchant Onboarding)
```
├── 收集KYC/KYB材料
│   ├── 营业执照
│   ├── 法人身份证明
│   ├── 银行账户信息
│   └── 业务证明材料
├── 提交至收单机构审批
├── MID (Merchant ID) 申请
└── 终端/网关配置
```

#### 4.3 设备与技术支持
```
├── POS终端销售/租赁
├── 设备安装与调试
├── 支付网关集成支持
└── 技术故障排查
```

#### 4.4 客户服务
```
├── 7x24 客服热线
├── 交易查询与对账
├── 退款/拒付处理协助
└── 费率调整申请
```

### 5. ISO 收入模式

#### 5.1 收入来源

```
┌─────────────────────────────────────────────────────────────┐
│                    商户支付的总手续费                         │
│                    (例如: 2.5%)                              │
└─────────────────────────────────────────────────────────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  Interchange  │   │  Assessment   │   │   Markup      │
│   (交换费)     │   │   (评估费)     │   │   (加价)      │
│   ~1.5-2.0%   │   │   ~0.1-0.15%  │   │   ~0.3-0.5%   │
│   → 发卡行     │   │   → 卡组织     │   │   → 分润      │
└───────────────┘   └───────────────┘   └───────────────┘
                                               │
                          ┌────────────────────┼────────────────────┐
                          │                    │                    │
                          ▼                    ▼                    ▼
                   ┌───────────┐        ┌───────────┐        ┌───────────┐
                   │ Acquirer  │        │    ISO    │        │   Agent   │
                   │   ~0.1%   │        │  ~0.15%   │        │  ~0.05%   │
                   └───────────┘        └───────────┘        └───────────┘
```

#### 5.2 收入类型

| 收入类型                | 说明                   | 典型金额            |
| ----------------------- | ---------------------- | ------------------- |
| **Residual Income**     | 交易流水分润（最主要） | 交易额的 0.05%-0.2% |
| **Upfront Bonus**       | 新商户签约奖金         | $50-$500/商户       |
| **Equipment Sales**     | 设备销售利润           | $50-$200/台         |
| **Monthly Fees**        | 月服务费分成           | $5-$25/商户/月      |
| **PCI Compliance Fees** | PCI合规费分成          | $5-$15/商户/月      |

#### 5.3 Residual 计算示例

```
假设:
- 商户月交易额: $100,000
- 商户费率: 2.5%
- Buy Rate (ISO成本): 2.2%
- ISO Spread: 0.3%

ISO 月收入 = $100,000 × 0.3% = $300/月

如果 ISO 有 500 个类似商户:
月 Residual = $300 × 500 = $150,000/月
```

### 6. 成为 ISO 的要求

#### 6.1 注册要求

```
Visa/Mastercard ISO 注册要求:
├── 与收单机构签署 ISO Agreement
├── 提交 ISO Registration Form
├── 背景调查 (Principal 背调)
├── 财务能力证明
├── 合规计划 (Compliance Program)
└── 年度注册费 ($5,000-$10,000)
```

#### 6.2 合规要求

| 要求                | 说明                              |
| ------------------- | --------------------------------- |
| **PCI DSS**         | 如处理/存储卡数据，需SAQ或ROC认证 |
| **AML/KYC**         | 反洗钱和客户身份识别程序          |
| **OFAC Screening**  | 制裁名单筛查                      |
| **Data Security**   | 数据安全和隐私保护                |
| **Agent Oversight** | 下级代理监管责任                  |

#### 6.3 资金要求

```
启动资金需求:
├── 注册费: $5,000-$10,000
├── 保证金: $25,000-$100,000 (视收单机构要求)
├── 运营资金: $50,000-$200,000
├── 设备库存: $20,000-$50,000
└── 营销预算: $10,000-$50,000
────────────────────────────────
总计: $110,000-$410,000
```

### 7. ISO vs PayFac 详细对比

| 维度             | ISO                    | PayFac               |
| ---------------- | ---------------------- | -------------------- |
| **商户合同关系** | 商户与收单机构直接签约 | 商户与PayFac签约     |
| **MID 归属**     | 每个商户独立MID        | 共享主商户MID        |
| **入驻时间**     | 3-7个工作日            | 分钟级               |
| **风险承担**     | 收单机构承担           | PayFac承担           |
| **资金流**       | 收单机构→商户          | 收单机构→PayFac→商户 |
| **技术投入**     | 低（使用现成平台）     | 高（自建系统）       |
| **启动成本**     | $100K-$400K            | $500K-$2M+           |
| **收入潜力**     | 中等                   | 高                   |
| **适合规模**     | 中小型商户服务         | 平台型业务           |

### 8. ISO 的挑战与风险

```
主要挑战:
├── 激烈的市场竞争
│   └── 费率战导致利润压缩
├── 商户流失 (Attrition)
│   └── 行业平均年流失率 15-25%
├── 合规成本上升
│   └── PCI DSS、隐私法规
├── 技术变革
│   └── 移动支付、BNPL冲击传统模式
└── 收单机构依赖
    └── 政策变化、费率调整风险
```

### 9. 国内对应模式

在中国支付市场，类似 ISO 的角色包括：

| 国际概念 | 国内对应               |
| -------- | ---------------------- |
| ISO      | 支付机构代理商/服务商  |
| Agent    | 二级代理、地推团队     |
| Acquirer | 银联商务、持牌支付机构 |

```
国内支付代理层级:
├── 一级代理 (类似 Wholesale ISO)
│   └── 直接与支付机构签约
├── 二级代理
│   └── 与一级代理签约
└── 地推团队
    └── 末端销售执行
```

### 10. 总结

**ISO 模式的核心价值主张：**

```
"低风险、低门槛进入支付行业，通过销售能力换取持续分润收入"
```

**适合人群：**
- 有商户资源但缺乏技术能力
- 希望稳定被动收入
- 不愿承担支付风险
- 专注于销售和客户服务
